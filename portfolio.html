NC-News Front-end

NC-News

As part of our front-end review at NorthCoders, we built the front-end application utilising my back-end API, over the course of our review week. This front-end allows users to view articles, rate them positively or negatively, make comments as a logged in user, delete their comments, and vote individual comments positively or negatively. The front-end provides the user with an interactive interface - allowing them to make HTTP methods to the back-end API.

The aim of this project was to bring together the front-end skills and technologies learnt during our front-end block at Northcoders, i.e. building an interactive user interface using ReactJs, and making asynchronous API calls to the server I had previously built. The project consolidates my overall understanding of CRUD applications.

The site is mobile-responsive, without sacrificing the style or functionality.


Tech-Stack
  • ReactJS - Javascript library for building interfaces.
  • Reach Router - A route manager for the react-app.
  • Axios - A promise based HTTP client for the browser and node.js.

NC-News API

NC-News

As part of our back-end review at NorthCoders, we were tasked with a one-week sprint to build a back-end API for a news app. The app allows users to read topics and articles, to post, update and delete comments, and to 'up-vote' and 'down-vote' articles and comments, in a similar fashion to reddit.

This is a RESTful API using all CRUD functions, built with full TDD best-practices and uses a PSQL database built using knex and express.

Tech-Stack
  • Node.js - Javascript runtime
  • PostgreSQL - open-source relational database system
  • Express - a web framework for Node.js
  • Knex - an SQL query builder
